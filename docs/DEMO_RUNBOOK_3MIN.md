# 3분 데모 런북

## 0:00 ~ 0:20 문제 정의

- 팀 프로젝트 평가에서 주관·무임승차 이슈가 발생함
- 본 시스템은 Rule + ML + AI 설명으로 평가 근거를 구조화함

## 0:20 ~ 1:20 핵심 데모

1. `팀 평가` 화면에서 데모 데이터 입력
2. `자동 평가 실행`
3. 결과에서 아래 순서로 설명
   - 정규화 점수, DL 보강 점수, 최종 혼합 점수
   - 이상 탐지(규칙)와 AI 설명
   - 데이터 신뢰도

## 1:20 ~ 2:20 심사기준 대응

- 심사기준 점수 요약(기술/AI/실무/창의) 확인
- `심사 제출 패키지` 버튼 클릭
- JSON 내 `judge_criteria_scores`, `judge_criteria_rationale`, `judge_comments` 확인

## 2:20 ~ 3:00 실무성/지속 개선

- 추세 그래프(30/60/90일 + 멤버별)로 장기 변화 설명
- DB 누적 저장 + 자동 재학습 + 모델 버전관리 소개
- 결론: 단발성 평가가 아니라 운영형 평가 보조 시스템
